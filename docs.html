FLICK — Gesture-Controlled Exam Platform

FLICK is a browser-native, AI-powered exam platform where students answer questions using hand gestures — no mouse, no keyboard, no extra hardware. One hand. One camera. Zero friction.

What is FLICK?

FLICK reimagines the exam experience by replacing the traditional mouse-and-keyboard interface with real-time hand gesture recognition. Using Google MediaPipe Hands running entirely in the browser, FLICK tracks 21 hand landmarks per frame and classifies them into 9 distinct gestures — each mapped to a specific exam action.

Questions are read aloud automatically via the Web Speech API, making FLICK inherently accessible. A host/student portal system with timed single-use access codes ensures each exam session is secure, proctored, and traceable.

Zero install, zero plugins FLICK runs entirely in a standard browser tab. No extensions, no apps, no external dependencies beyond an internet connection for the MediaPipe model.

Who is it for?

Teachers / Hosts
  • Build multi-question exam forms with a visual builder
  • Mark correct answers per question
  • Generate per-student timed access codes
  • Monitor live session validity in the Sessions panel
  • View and download results with full answer keys
Students

How it works

FLICK is a single-file HTML application. Every feature — gesture detection, speech synthesis, proctoring, storage, and reporting — runs client-side with no backend required.

Gesture pipeline

Each video frame from the webcam is sent to the MediaPipe Hands model, which returns 21 3D landmarks for the detected hand. FLICK's classifier reads those landmarks and maps them to one of 9 gesture labels. To prevent false triggers, labels are pushed into a 10-frame rolling buffer — a gesture only fires when 70% or more of recent frames agree on the same label.

javascript — gesture classifier 
function classifyLM(lm) {
  // tip-above-pip check with 0.03 threshold gap
  function ext(tip, pip) {
    return lm[tip].y < lm[pip].y - 0.03;
  }
  const thumbExt = Math.abs(lm[4].x - lm[2].x) > 0.05;
  const [i, m, r, p] = [ext(8,6), ext(12,10), ext(16,14), ext(20,18)];

  if (lm[4].y < lm[3].y - 0.05 && !i && !m && !r && !p) return 'next';
  if (lm[4].y > lm[3].y + 0.05 && !i && !m && !r && !p) return 'prev';
  if (p && !i && !m && !r && !thumbExt)          return 'speak';
  if (thumbExt && i && m && r && p)              return 'submit';
  if (!thumbExt && !i && !m && !r && !p)         return 'confirm';
  if (i && !m && !r && !p)                       return 'opt0';
  if (i && m && !r && !p)                        return 'opt1';
  if (i && m && r && !p)                         return 'opt2';
  if (i && m && r && p && !thumbExt)             return 'opt3';
}

Dwell-to-select timing

Rather than triggering on a single frame, every gesture requires a dwell period — the gesture must be held continuously for the action to fire. This eliminates accidental triggers from passing hand movements.

Gesture system

FLICK recognises 9 distinct hand gestures. Each is defined by a unique combination of extended fingers, making them mutually exclusive — no gesture can be confused for another.

☝️ 1 finger Index finger only extended opt0 → Option A
✌️ 2 fingers Index + middle extended opt1 → Option B
🤟 3 fingers Index + middle + ring opt2 → Option C
🖐️ 4 fingers All fingers except thumb opt3 → Option D
✊ Fist No fingers extended confirm → instant select
👍 Thumb up Thumb above index, rest closed next → next question
👎 Thumb down Thumb below index, rest closed prev → previous
🤙 Pinky only Pinky extended, others closed speak → read aloud
🖖 All 5 fingers Thumb + all 4 fingers open submit → hold 3s
Tips for best recognition Face your palm toward the camera. Keep your hand well-lit and within frame. The stability meter in the cam panel shows real-time confidence — aim for 70%+ (green bars) before holding a gesture.

Access codes

Every exam session is gated by a unique 6-character alphanumeric access code generated by the host. Codes are designed to be unambiguous — the character set excludes visually similar characters.

Character set

javascript
// Excludes: O, 0, I, 1 (visually ambiguous)
const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789';

function genCode() {
  let c = '';
  for (let i = 0; i < 6; i++)
    c += chars[Math.floor(Math.random() * chars.length)];
  return c; // e.g. "A3KZ9P"
}

Code lifecycle

StateConditionStudent can enter?
pendingGenerated, not yet used, within 20 minYes
activeStudent has passed validation, exam runningNo (in use)
usedExam submitted (any reason)No (consumed)
expired20 minutes elapsed without useNo (expired)
Codes are strictly single-use Once a student submits (for any reason — normal submit, tab switch, or timeout), the code is immediately invalidated and its expiry is set to Date.now(). It cannot be re-entered.

Anti-cheat

FLICK includes two layers of proctoring that work together to keep exams honest.

Tab-switch detection

FLICK listens to two browser events simultaneously. If either fires while the exam overlay is visible, a 3-second countdown overlay appears and the exam auto-submits with a reason: "tab-switch" flag on the result.

javascript
// Fires when student switches to another tab
document.addEventListener('visibilitychange', () => {
  if (document.hidden) triggerTabViolation();
});

// Fires when student clicks away (another window, OS, etc.)
window.addEventListener('blur', () => {
  triggerTabViolation();
});

Permission-first flow

Camera and microphone permissions are requested on a dedicated screen before the exam timer starts. This prevents the browser's permission popup from stealing window focus and accidentally triggering the tab-switch guard.

Stream reuse Once permissions are granted, the media stream is passed directly into the exam — no second permission dialog ever fires, even as MediaPipe initialises.

Auto text-to-speech

Every question is read aloud automatically 3 seconds after it appears on screen. A progress bar in the cam panel counts down visibly. Students can re-trigger reading at any time with the pinky gesture, or press the Read aloud button.

javascript — auto-read timer
function startAutoRead() {
  let elapsed = 0;
  const DURATION = 3000; // 3 seconds

  EX.autoReadTimer = setInterval(() => {
    elapsed += 100;
    // update progress bar ...
    if (elapsed >= DURATION) {
      cancelAutoRead();
      speakQ(false); // auto-triggered, not manual
    }
  }, 100);
}

The auto-read timer is cancelled any time the student navigates to a different question — whether by gesture or the on-screen navigator dots. A fresh 3-second countdown begins for the new question.

Host portal

The host portal has four tabs: My Exams, Builder, Codes, and Results.

Form builder

The visual builder lets you add up to 20 questions per form. Each question requires exactly 4 options with one correct answer marked. Forms are saved to localStorage and persist across browser sessions.

Storage note All data (forms, sessions, results) is stored in localStorage in the current version. This means data is per-device and per-browser. For shared multi-device access, a Firebase backend is recommended — see the deployment section.

Sessions panel

The Codes tab shows all generated sessions with live countdown timers, status badges (Live, Expired, Completed), and a colour-coded validity bar that turns amber below 50% and red below 20%.

Student portal

Exam flow

1

Enter code

Student enters their name and the 6-character code from email. The code is validated instantly against the session store — checking expiry and single-use status before proceeding.

2

Grant permissions

A dedicated setup screen requests camera and microphone access. The exam timer does not start until the student clicks Start Exam Now — granting permissions never costs exam time.

3

Answer by gesture

Questions appear full-screen with the cam panel on the left. Each question reads itself aloud after 3 seconds. The student holds a gesture for 1.5 seconds to select an option, or uses thumb-up to advance.

4

Submit

Hold all 5 fingers open for 3 seconds to submit, or navigate past the last question. The exam also auto-submits on tab switch or timer expiry.

5

View result

The result screen shows score percentage, correct/wrong/skipped counts, and a full question-by-question review with the answer key. The report can be downloaded as a standalone HTML file.

Reports

FLICK generates a complete exam report the moment an exam is submitted. The report is available to both the student (immediately) and the host (via the Results tab).

Downloadable report

Clicking Download Report generates a standalone .html file containing the full result including the answer key — correct answers highlighted green, wrong answers highlighted red. This file can be printed or archived without needing FLICK to be open.

FAQ

Does FLICK work on mobile?

FLICK is responsive and loads on mobile browsers, but gesture recognition is optimised for front-facing laptop cameras. Mobile performance depends heavily on camera quality and frame rate. We recommend desktop Chrome for the best experience.

What happens if the student's camera is blocked?

If camera access is denied, the gesture cam panel shows a No camera message. The exam can still be taken using the on-screen buttons — Next, Read aloud, and the option cards are all clickable.

Can a host take their own exam?

Yes — a host can switch to the Student portal from the same device and enter a code they generated. Sessions are stored in localStorage which is shared across the same browser.

Can the same code be used twice?

No. Codes are strictly single-use. Once an exam is submitted for any reason, the code status is set to used and the expiry is immediately set to Date.now(). A second attempt with the same code will show an "already used" error.

Does FLICK send data to any server?

In the default deployment, all data lives in localStorage on the user's device — nothing is sent to any server. The MediaPipe model runs entirely client-side. If you add EmailJS, code metadata is sent to EmailJS servers only for the purpose of delivering the email.
